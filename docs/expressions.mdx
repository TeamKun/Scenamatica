# 用語集

Scenamatica で使用される用語と一般的な用語をまとめています。

---

## Scenamatica で使用される用語 {#scenamatica-expressions}

### シナリオファイル {#scenario-file}

シナリオファイルは、テストする[シナリオ](#scenario)を記述したファイルです。

シナリオファイルは、以下の要素を内包します：
+ [シナリオ](#scenario)の名前
+ [シナリオのトリガ](#trigger)
+ [シナリオのコンテキスト](#context)
+ 本[シナリオ](#scenario)

### トリガ {#trigger}

トリガは、シナリオが実行されるタイミングを指定するための要素です。
[シナリオファイル](#scenario-file)に複数のトリガを指定しておき、
いずれかが満たされた場合に本番[シナリオ](#scenario)が実行されます。

関連項目： [事前シナリオ](#before-scenario)、[事後シナリオ](#after-scenario)

### コンテキスト {#context}

コンテキストは、[シナリオ](#scenario)を実行するために必要な情報を指定するための要素です。
コンテキストには以下の２つの要素があります：

#### アクター {#context-actor}

アクターは、[シナリオ](#scenario)に登場する Minecraft プレイヤです。
[シナリオ](#scenario)開始時には指定された名前でログインします。また[シナリオ](#scenario)が終了すると関連する全ての情報が削除され、抹消されます。

#### ステージ {#context-stage}

ステージは、[シナリオ](#scenario)に登場する Minecraft ワールドです。
[シナリオ](#scenario)開始時に生成されます。また[シナリオ](#scenario)が終了すると削除されます。
[シナリオ](#scenario)上で行われたステージ上の変更は全てロールバックされます。

### シナリオ {#scenario}

シナリオは、テストの流れを記述した要素です。
上から順に実行され、途中で失敗した場合はテスト自体が失敗となり、以降の処理は実行されません。

### 事前シナリオ {#before-scenario}

事前シナリオは、特定の[トリガ](#trigger)において、本[シナリオ](#scenario)の実行前に実行される[シナリオ](#scenario)です。

### 事後シナリオ {#after-scenario}

事後シナリオは、特定の[トリガ](#trigger)において、本[シナリオ](#scenario)の実行後に実行される[シナリオ](#scenario)です。
